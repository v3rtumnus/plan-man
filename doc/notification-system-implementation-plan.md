# Notification System — Implementation Plan

## Overview

This module replaces the hard-coded Lorem Ipsum notification stub in `fragments/header.html` with a
fully functional, database-backed notification system. Notifications are generated by configurable
**rules**, each evaluated on a schedule. A rule encodes a condition in a specific domain (e.g.
expenses vs. liquid budget) plus typed parameters (e.g. threshold amount). Evaluators are Spring
beans — adding a new rule type is a single new class implementing one interface.

**First concrete rule:** *Notify when the actual expenses for the current month come within
`thresholdAmount` EUR of the monthly liquid budget (configurable, default 300 EUR).*

The liquid budget for a month is calculated the same way `BalanceService.getBalanceComparisons()`
does it: sum of income `BalanceItemDetail` entries minus sum of expenditure entries active on the
1st of the month (`balanceDetailRepository.getRelevantAmountsForDate`). The actual expenses are the
sum of all `Expense` rows for that year/month from `ExpenseService`.

it fans out only to the configured usernames. This gives each user their own independent read state.

### 2.1 Create `service/notification/NotificationEvaluator.java`

```java
public interface NotificationEvaluator {

    /** The rule type this evaluator handles. */
    NotificationRuleType getRuleType();

    /**
     * Evaluate the rule and return a result describing whether a notification
     * should be created. Returns empty Optional if the condition is not met.
     *
     * @param rule the rule to evaluate (parameters available via getParametersJson())
     * @return Optional containing the notification to persist, or empty if condition is not met
     */
    Optional<NotificationCandidate> evaluate(NotificationRule rule);

    /** Simple value object: title, message, severity, periodKey. */
    record NotificationCandidate(String title, String message,
                                 NotificationSeverity severity, String periodKey) {}
}
```

All evaluators are Spring `@Component` beans. `NotificationRuleEvaluationService` collects them
by type using a `Map<NotificationRuleType, NotificationEvaluator>` built in `@PostConstruct`.

### 2.2 Create `service/notification/evaluator/ExpensesNearLiquidThresholdEvaluator.java`

**Rule parameters JSON:** `{ "thresholdAmount": 300.00 }`

**Logic:**
1. Parse `thresholdAmount` from `rule.getParametersJson()` using `ObjectMapper`
2. Get `today = LocalDate.now()`; `periodKey = "YYYY-MM"` (e.g. `"2026-03"`)
3. Compute `liquidBudget`:
   - Call `balanceDetailRepository.getRelevantAmountsForDate(today.withDayOfMonth(1))`
   - Sum INCOME entries positively, EXPENDITURE entries negatively (same logic as `BalanceService`)
4. Compute `currentExpenses`:
   - Call `expenseService.getExpensesForMonth(today.getYear(), today.getMonthValue())`
   - Sum all `ExpenseDTO.getAmount()` values
5. Compute `remaining = liquidBudget - currentExpenses`
6. If `remaining < thresholdAmount`:
   - Return `NotificationCandidate` with:
     - title: `"Budget-Warnung: Nur noch X EUR verfügbar"`
     - message: `"Die Ausgaben für [Monat Jahr] betragen X EUR. Das monatliche Budget beträgt Y EUR. Es verbleiben nur noch Z EUR bis zur Budgetgrenze."`
     - severity: `WARNING` (if `remaining > 0`), `DANGER` (if `remaining <= 0`)
     - periodKey: `"2026-03"` (prevents duplicate for same month)
7. Else return `Optional.empty()`

## Step 3 — Notification Service

### 3.1 Create `service/notification/NotificationService.java`

Responsibilities: notification CRUD, header data aggregation, read state management.

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class NotificationService {

    private final NotificationRepository notificationRepository;
    private final NotificationRuleRepository ruleRepository;
    private final ObjectMapper objectMapper;

    /** Used by GlobalController — injected into every page. Scoped to the logged-in user. */
    public NotificationHeaderDTO getHeaderData(String username) {
        long unreadCount = notificationRepository.countByRecipientUsernameAndReadFalse(username);
        List<NotificationDTO> recent = notificationRepository
            .findTop5ByRecipientUsernameAndReadFalseOrderByCreatedAtDesc(username)
            .stream().map(this::toDTO).toList();
        return new NotificationHeaderDTO((int) unreadCount, recent);
    }

    /** Full list for the notifications overview page. Scoped to the logged-in user. */
    public List<NotificationDTO> findAll(String username) {
        return notificationRepository.findAllByRecipientUsernameOrderByReadAscCreatedAtDesc(username)
            .stream().map(this::toDTO).toList();
    }

    public void markAsRead(Long id, String username) {
        Notification n = notificationRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("Notification not found: " + id));
        if (!n.getRecipient().getUsername().equals(username)) {
            throw new RuntimeException("Notification does not belong to user: " + username);
        }
        n.setRead(true);
        notificationRepository.save(n);
    }

    public void markAllRead(String username) {
        notificationRepository.markAllReadForUser(username);
    }

    public void delete(Long id, String username) {
        Notification n = notificationRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("Notification not found: " + id));
        if (!n.getRecipient().getUsername().equals(username)) {
            throw new RuntimeException("Notification does not belong to user: " + username);
        }
        notificationRepository.delete(n);
    }

    /**
     * Called by the evaluation service to persist a notification for a specific user.
     * Deduplication is per (rule, username, periodKey).
     */
    public void createNotification(NotificationRule rule, UserProfile recipient,
                                   NotificationEvaluator.NotificationCandidate candidate) {
        if (notificationRepository.existsByRuleAndRecipientUsernameAndPeriodKey(
                rule, recipient.getUsername(), candidate.periodKey())) {
            log.debug("Skipping duplicate notification for rule '{}', user '{}', period '{}'",
                rule.getName(), recipient.getUsername(), candidate.periodKey());
            return;
        }
        notificationRepository.save(Notification.builder()
            .rule(rule)
            .recipient(recipient)
            .title(candidate.title())
            .message(candidate.message())
            .severity(candidate.severity())
            .read(false)
            .createdAt(LocalDateTime.now())
            .periodKey(candidate.periodKey())
            .build());
        log.info("Created notification '{}' for rule '{}', user '{}'",
            candidate.title(), rule.getName(), recipient.getUsername());
    }

    // CRUD for rules (used by admin controller)
    public List<NotificationRuleDTO> findAllRules() { ... }
    public NotificationRuleDTO createRule(NotificationRuleDTO dto) { ... }
    public NotificationRuleDTO updateRule(Long id, NotificationRuleDTO dto) { ... }
    public void deleteRule(Long id) { ... }
    public void setRuleEnabled(Long id, boolean enabled) { ... }
}
```

### 3.2 Create `service/notification/NotificationRuleEvaluationService.java`

Runs all enabled rules on a configurable schedule. Delegates to the appropriate evaluator per rule
type.

```java
@Service
@RequiredArgsConstructor
@Slf4j
public class NotificationRuleEvaluationService {

    private final NotificationRuleRepository ruleRepository;
    private final NotificationService notificationService;
    private final UserProfileRepository userProfileRepository;
    private final List<NotificationEvaluator> evaluators;  // Spring injects all implementations

    private Map<NotificationRuleType, NotificationEvaluator> evaluatorMap;

    @PostConstruct
    public void init() {
        evaluatorMap = evaluators.stream()
            .collect(Collectors.toMap(NotificationEvaluator::getRuleType, e -> e));
    }

    @Scheduled(cron = "${notifications.evaluation.cron:0 0 * * * *}")
    public void evaluateAll() {
        log.info("Starting notification rule evaluation");
        ruleRepository.findByEnabledTrue().forEach(this::evaluateRule);
    }

    public void evaluateRule(NotificationRule rule) {
        NotificationEvaluator evaluator = evaluatorMap.get(rule.getRuleType());
        if (evaluator == null) {
            log.warn("No evaluator found for rule type: {}", rule.getRuleType());
            return;
        }
        try {
            evaluator.evaluate(rule).ifPresent(candidate -> fanOut(rule, candidate));
        } catch (Exception e) {
            log.error("Error evaluating rule '{}': {}", rule.getName(), e.getMessage(), e);
        }
    }

    /**
     * Fan-out: creates one notification row per target user so that each user has
     * their own independent read/unread state.
     * - ALL_USERS  → all rows in user_profile
     * - SPECIFIC_USERS → only the users listed in rule.getTargetUsers()
     */
    private void fanOut(NotificationRule rule, NotificationEvaluator.NotificationCandidate candidate) {
        List<UserProfile> targets = switch (rule.getTargetingMode()) {
            case ALL_USERS      -> userProfileRepository.findAll();
            case SPECIFIC_USERS -> rule.getTargetUsers();
        };
        targets.forEach(user -> notificationService.createNotification(rule, user, candidate));
    }

    /** Manual trigger: evaluate a single rule by ID (used by admin "Run Now" action). */
    public void evaluateById(Long ruleId) {
        ruleRepository.findById(ruleId).ifPresent(this::evaluateRule);
    }
}
```

## Step 5 — API Controller

### 5.1 Create `controller/api/NotificationApiController.java`

```java
@RestController
@RequestMapping("/api/notifications")
@RequiredArgsConstructor
public class NotificationApiController {

    private final NotificationService notificationService;
    private final NotificationRuleEvaluationService evaluationService;

    // --- Notifications (all scoped to the authenticated user) ---
    @GetMapping
    public List<NotificationDTO> list(Authentication auth) {
        return notificationService.findAll(auth.getName());
    }

    @PostMapping("/{id}/read")
    public ResponseEntity<Void> markRead(@PathVariable Long id, Authentication auth) {
        notificationService.markAsRead(id, auth.getName());
        return ResponseEntity.ok().build();
    }

    @PostMapping("/read-all")
    public ResponseEntity<Void> markAllRead(Authentication auth) {
        notificationService.markAllRead(auth.getName());
        return ResponseEntity.ok().build();
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id, Authentication auth) {
        notificationService.delete(id, auth.getName());
        return ResponseEntity.noContent().build();
    }

    // --- Rules ---
    @GetMapping("/rules")
    public List<NotificationRuleDTO> listRules() { return notificationService.findAllRules(); }

    @PostMapping("/rules")
    public ResponseEntity<NotificationRuleDTO> createRule(@RequestBody NotificationRuleDTO dto) {
        return ResponseEntity.status(HttpStatus.CREATED).body(notificationService.createRule(dto));
    }

    @PutMapping("/rules/{id}")
    public NotificationRuleDTO updateRule(@PathVariable Long id, @RequestBody NotificationRuleDTO dto) {
        return notificationService.updateRule(id, dto);
    }

    @DeleteMapping("/rules/{id}")
    public ResponseEntity<Void> deleteRule(@PathVariable Long id) {
        notificationService.deleteRule(id);
        return ResponseEntity.noContent().build();
    }

    @PostMapping("/rules/{id}/enable")
    public ResponseEntity<Void> enable(@PathVariable Long id) {
        notificationService.setRuleEnabled(id, true);
        return ResponseEntity.ok().build();
    }

    @PostMapping("/rules/{id}/disable")
    public ResponseEntity<Void> disable(@PathVariable Long id) {
        notificationService.setRuleEnabled(id, false);
        return ResponseEntity.ok().build();
    }

    @PostMapping("/rules/{id}/run")
    public ResponseEntity<Void> runNow(@PathVariable Long id) {
        evaluationService.evaluateById(id);
        return ResponseEntity.accepted().build();
    }
}
```

## Concrete Data Flow — Expenses Near Liquid Threshold

```
Top of the hour: @Scheduled fires (NotificationRuleEvaluationService.evaluateAll)
     │
     ▼  Loads all enabled rules from DB
     │   → finds rule: EXPENSES_NEAR_LIQUID_THRESHOLD { thresholdAmount: 300.00 }
     │
     ▼  Dispatches to ExpensesNearLiquidThresholdEvaluator.evaluate(rule)
     │
     ▼  Reads balanceDetailRepository.getRelevantAmountsForDate(2026-03-01)
     │   → income items: Gehalt 3200, Miete-Einnahme 500 = 3700 EUR
     │   → expenditure items: Miete -1200, Strom -80 = -1280 EUR
     │   → liquidBudget = 3700 - 1280 = 2420 EUR
     │
     ▼  Reads expenseService.getExpensesForMonth(2026, 3)
     │   → actual expenses so far: 2180 EUR
     │
     ▼  remaining = 2420 - 2180 = 240 EUR
     │   240 < thresholdAmount (300) → condition MET
     │
     ▼  Returns NotificationCandidate:
     │   title:     "Budget-Warnung: Nur noch 240 EUR verfügbar"
     │   message:   "Die Ausgaben für März 2026 betragen 2.180 EUR. Das monatliche
     │               Budget beträgt 2.420 EUR. Es verbleiben nur noch 240 EUR."
     │   severity:  WARNING
     │   periodKey: "2026-03"
     │
     ▼  fanOut(rule, candidate) — rule.targetingMode = ALL_USERS
     │   → userProfileRepository.findAll() → [alice, bob]
     │   → calls createNotification(rule, alice, candidate)
     │       existsByRuleAndRecipientUsernameAndPeriodKey(rule, "alice", "2026-03") → false
     │       persists Notification(recipient=alice, read=false)
     │   → calls createNotification(rule, bob, candidate)
     │       existsByRuleAndRecipientUsernameAndPeriodKey(rule, "bob", "2026-03") → false
     │       persists Notification(recipient=bob, read=false)
     │
     ▼  Next page load for alice: GlobalController injects notificationHeader("alice")
         → unreadCount = 1, notifications = [alice's notification]
         → header bell badge shows "1", dropdown shows the budget warning
         → bob's bell also shows "1" independently; alice reading hers does not affect bob's
```

Subsequent hourly runs for the same month: `existsByRuleAndRecipientUsernameAndPeriodKey` returns
`true` for each user → skipped, no duplicates created.

## Extensibility — Adding a New Rule Type

1. Add enum value to `NotificationRuleType`
2. Create `service/notification/evaluator/YourNewEvaluator.java` implementing `NotificationEvaluator`
3. Spring auto-discovers it; `NotificationRuleEvaluationService` maps it by type at startup
4. Add the parameters JSON schema to the reference section in `rules.html`
5. Add a DB row for the new rule (or let the user create it via the admin UI)

No changes to any existing class are required.
